# Inline

## Use case

The `Inline` component is designed to create consistent spacing around inline elements of variable width. Unlike the `InlineCluster` component,
the items in the `Inline` component will not wrap. The `Inline` component also allows you to specify a component that can stretch and fill the excess space.

## Installation

```bash
yarn add @bedrock-layout/inline
```

## InlineCluster Props

The `Inline` component has all the same props as the `InlineCluster` component. (check the [InlineCluster component for more details](/?path=/docs/components-inlinecluster--api))

## minItemWidth

Some times you want all the items to have a minimum width. By setting the `minItemWidth` prop, you will set the minimum width of each of the children.

This is especially useful with the `switchAt` prop (see below).

In the below example, each of the Componentes has an intrinsic width of 50px. The `minItemWidth` will make them `150px` each.

<Canvas withSource="open">
  <Story name="minItemWidth">
    <Inline gutter="size3" minItemWidth={150}>
      <Component style={{}} />
      <Component />
      <Component />
      <Component />
    </Inline>
  </Story>
</Canvas>

## switchAt

In the below example, the layout will switch to stack when its width becomes less than `45rem`. (Resize your window to see this in action.)

<Canvas withSource="open">
  <Story name="switchAt">

      <Component />
      <Component />
      <Component />
      <Component />
    </Inline>
  </Story>
</Canvas>

## data-attribute

For styling purposes, you can select `data-bedrock-inline`.
