## <Intl>Contributing Code or Documentation</Intl>

<p>
  <Intl>
    Bedrock Layout Primitives is a monorepo of packages that are currently
    versioned independently of each other. It also utilizes Storybook as code
    playground, a documentation website, and visual regression test runner in
    chromatic.
  </Intl>
</p>

<p>
  <Intl>To manage all of these things, Bedrock uses </Intl>
  <a href="https://lerna.js.org" target="_blank" rel="noreferer">
    Lerna
  </a> <Intl>to manage all the repositories into one monorepo. Lerna is used to publish to NPM, tag releases to Github, and process change logs under each package. To make the above possible in CI, Bedrock has adopted conventional commits and uses commitizen and husky to enforce it.</Intl>
</p>

<p>
  <Intl>To make a pull request, do the following:</Intl>
</p>

<ol>
  <li>
    <Intl>Fork the repository.</Intl>
  </li>
  <li>
    <Intl>Clone the repository from your GitHub.</Intl>
  </li>
  <li>
    <Intl>Set up your environment with yarn</Intl>
  </li>
  <li>
    <Intl>Check out a new branch and add your modification.</Intl>
  </li>
  <li>
    <Intl>Add test cases for all your changes.</Intl>
  </li>
  <li>
    <Intl>
      Update any README.md and documentation in Storybook under the examples
      folder in the corresponding package.
    </Intl>
  </li>
  <li>
    <Intl>
      Use commitizen to do git commit by running yarn cz (be sure to stage your
      changes first). Doing this will automatically run yarn verify as a
      pre-commit hook when you do this. If anything fails, fix and follow this
      step again.
    </Intl>
  </li>
  <li>
    <Intl>**DO NOT BUMP ANY VERSION NUMBERS OR UPDATE CHANGELOG.**</Intl>
  </li>
  <li>
    <Intl>Send a pull request üôè</Intl>
  </li>
</ol>
