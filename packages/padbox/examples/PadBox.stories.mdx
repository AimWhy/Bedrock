# PadBox

## Use case

The `PadBox` component is designed to create consistent padding based on the spacing constants.
PadBox takes either a single value, an array of values (like the css shorthand for top / right / bottom / left), or an object of values (specifying each side individually) for fine tuning the padding.

Padding values can either be `SpacingOptions` defined in the theme (or the default bedrock `SpacingOptions`), a positive `number` of pixels, or a valid `CSSLength`. If you provided an invalid value (such as a negative number), the padding will be set to `0px`.

Some examples of where you might use a `PadBox`:

- [Product Card](/docs/examples-basic--docs#product-card)
- [Basic Form](/docs/examples-basic--docs#basic-form)
- [Header Sections](/docs/examples-basic--docs#header-sections)
- [Aspect Ratio Image Card](/docs/examples-web-dev--docs#ratio-image-card)

## Installation

```bash
yarn add @bedrock-layout/padbox
```

## Padding as an object

The `padding` prop can also take an object to specify which locations will have padding and of which type.
You can pass either traditional properties like `top, bottom, right, left`, or logical properties in camelcase such as `blockStart, blockEnd, inlineStart, inlineEnd`.
No matter which properties are given, logical properties are used.
