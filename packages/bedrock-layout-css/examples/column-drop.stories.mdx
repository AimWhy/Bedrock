# column-drop.css

This is a CSS Only version of the [ColumnDrop package](/?path=/docs/components-columndrop--api).

## Use case

The ColumnDrop component is used to create a layout of columns that stretch to fit the space, and snaps to the next line at a minimum size.

## Usage

First you need to import the appropriate stylesheet. The grid is part of the main CSS bundle or you can import it in individually from the components directory.

```javascript
import '@bedrock-layout/bedrock-layout-css/lib/bedrock-layout.css';

// or

import "@bedrock-layout/bedrock-layout-css/lib/components/column-drop.css";
```

Then you add the `data-bedrock-column-drop` attribute to the desired element.

## --gutter

The `--gutter` prop defines the gutter size between elements. Bedrock has implemented a [default spacing scheme](css-only-components-spacing-properties-css--page), or you can simply pass any valid CSS length or percentage value.

You can adjust the space between the child elements by setting the `--gutter` property in the inline style.

## --minItemWidth

The grid will automatically optimize the number of columns based on the width of the container and the value provided to the `--minItemWidth` property. By default this is set to `159px`, but you can change this value by setting the `--minItemWidth` property in the style.

In the below example, The `--minItemWidth` is set to `15rem`. As you resize the window, the Grid will recalculate and potentially change the count of columns and rows.

## no-stretched-columns

By default, the column-drop component will stretch the columns to fit the container. If you want to prevent this behavior, you can set the `no-stretched-columns` property on the `data-bedrock-column-drop` attribute.

In the below example, The `--basis` is set to `15rem`. As you resize the window, the Grid will recalculate and potentially change the count of columns and rows.
